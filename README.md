> **Live Site:** [vinodkalharaportfolio.netlify.app](https://vinodkalharaportfolio.netlify.app/)

A modern, responsive, and highly interactive personal portfolio website built to showcase my journey as a Software Engineering undergraduate bridging the gap between Full-Stack Development and Artificial Intelligence. 

## ✨ Features

- **Dark-Themed Modern UI:** Glassmorphism effects, terminal-inspired design accents, and smooth micro-interactions.
- **Bento Grid Layouts:** Clean, structured display of skills, statistics, and connection links.
- **Fully Responsive:** Optimized for mobile, tablet, and desktop viewing.
- **Live Contact Form:** Integrated with Web3Forms for secure, direct-to-inbox messaging with spam protection.
- **Dynamic Projects Showcase:** Highlighting real-world applications of AI, backend engineering, and UI/UX design.

## 🛠️ Built With

This portfolio itself is a static site engineered for speed and sleek aesthetics:
- **HTML5 & CSS3** (Tailwind CSS for rapid utility-first styling)
- **Vanilla JavaScript** (ES6+)
- **Web3Forms API** (For seamless contact form handling)
- **Netlify** (For automated continuous deployment)

## 💻 Featured Work Highlighted

The portfolio features deep dives into my core projects:
1. **AI Support Agent:** An autonomous 24/7 email triage agent using n8n, OpenAI, and Python to classify inquiries and route tickets.
2. **Personal AI Knowledge Assistant:** A Python web application utilizing Retrieval-Augmented Generation (RAG) for context-aware, hyper-specific semantic search.
3. **GlowHive Skincare App:** A full-stack mobile platform (React Native, Node.js, MongoDB) featuring interactive diaries and product management.

## 🚀 Run Locally

If you want to clone and run this site on your local machine:

1. Clone the repository:
   ```bash
   git clone [https://github.com/it24104383Kalhara/vinod-portfolio.git](https://github.com/it24104383Kalhara/vinod-portfolio.git)
  
2. Navigate to the project directory:
   ```bash
   cd vinod-portfolio

3. Open index.html in your browser, or start a local server (recommended for module scripts):
    ```bash
    npx serve .
    ```
